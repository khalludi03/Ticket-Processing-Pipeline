# এআই-চালিত সাপোর্ট টিকিট প্রসেসিং পাইপলাইন - প্রজেক্ট ডকুমেন্টেশন (বাংলা)

## ১. প্রজেক্টের সংক্ষিপ্ত পরিচয়
এই প্রজেক্টটি একটি অটোমেটেড ব্যাকএন্ড সার্ভিস যা কাস্টমার সাপোর্ট টিকিটগুলোকে দুটি ধাপে এআই দিয়ে প্রসেস করে:
- **ট্রায়েজ (Triage)**: টিকিটের ক্যাটাগরি, প্রায়োরিটি, সেন্টিমেন্ট ইত্যাদি ক্লাসিফাই করা
- **রেজোলিউশন (Resolution)**: কাস্টমারের জন্য রিপ্লাই ড্রাফ্ট এবং ইন্টারনাল নোট তৈরি করা

প্রজেক্টটি অ্যাসিনক্রোনাস কিউ-বেসড প্রসেসিং, রিলেশনাল ডেটাবেস স্টেট ট্র্যাকিং এবং রিয়েল-টাইম ওয়েবসকেট নোটিফিকেশন ব্যবহার করে তৈরি করা হয়েছে।

## ৩. ডেটা ফ্লো - কিভাবে টিকিট প্রসেস হয়
একটি টিকিট সাবমিট করার পর কিভাবে সিস্টেমে প্রসেস হয় তার ধাপ:

1. **টিকিট সাবমিট**: `POST /tickets` এন্ডপয়েন্টে টিকিট ডেটা পাঠানো হয় → অথেনটিকেশন এবং রেট লিমিট চেক → ডেটাবেসে টিকিট সেভ (`status: queued`) → SQS কিউতে মেসেজ পাঠানো হয়
2. **ট্রায়েজ ধাপ**: কিউ কনজিউমার SQS থেকে মেসেজ নেয় → `status: processing` সেট করা হয় → এআই দিয়ে ট্রায়েজ অ্যানালাইসিস → ডেটাবেসে `triage_output` সেভ → রেজোলিউশন ধাপের জন্য নতুন কিউ মেসেজ পাঠানো হয়
3. **রেজোলিউশন ধাপ**: কিউ থেকে রেজোলিউশন মেসেজ নেয় → এআই দিয়ে রিপ্লাই ড্রাফ্ট তৈরি → `resolution_output` সেভ → `status: completed` সেট → ওয়েবসকেটের মাধ্যমে ক্লায়েন্টকে নোটিফাই করা হয়
4. **এরর হ্যান্ডলিং**: কোনো ধাপে এরর হলে ৩ বার রিট্রাই হয় (এক্সপোনেনশিয়াল ব্যাকঅফ) → রিট্রাই শেষ হলে `status: needs_manual_review` সেট করা হয় → ফলব্যাক আউটপুট সেভ করা হয়

## ৫. কিভাবে প্রজেক্টটি পেশাদারভাবে পড়বেন
প্রজেক্টটি বুঝতে নিচের ধাপগুলো অনুসরণ করুন:

1. **প্রথমে রুট ফাইলগুলো পড়ুন**: `package.json` → `tsconfig.json` → `CLAUDE.md` → `README.md` → `ticket-pipeline-architecture.md`। এতে প্রজেক্টের স্ট্রাকচার এবং নিয়মকানুন বুঝতে পারবেন।
2. **এন্ট্রি পয়েন্ট পড়ুন**: `src/index.ts` → অ্যাপ কিভাবে স্টার্ট হয়, রাউট কিভাবে অ্যাড হয়, কিউ কনজিউমার এবং ওয়েবসকেট কিভাবে ইনিশিয়ালাইজ হয় তা বুঝতে পারবেন।
3. **রাউট থেকে শুরু করুন**: `src/routes/tickets.ts` → কোন এন্ডপয়েন্টগুলো আছে, কোন মিডলওয়্যার ব্যবহার করা হয়েছে তা দেখুন।
4. **কন্ট্রোলার → সার্ভিস → রিপোজিটরি**: `src/controllers/tickets.controller.ts` → `src/services/tickets.service.ts` → `src/repositories/tickets.repository.ts`। এই লেয়ারড আর্কিটেকচার বুঝুন。
5. **এআই প্রসেসিং**: `src/ai/triage.ts` → `src/ai/resolution.ts` → দুই ধাপের এআই লজিক বুঝুন。
6. **কিউ এবং রিয়েলটাইম**: `src/queue/consumer.ts` → `src/realtime/` → অ্যাসিনক্রোনাস প্রসেসিং এবং ওয়েবসকেট নোটিফিকেশন বুঝুন。
7. **ডেটাবেস**: `src/db/schema.ts` → টেবিল স্ট্রাকচার এবং রিলেশন বুঝুন。
8. **টেস্ট**: `tests/` ফোল্ডার → ইউনিট এবং ইন্টিগ্রেশন টেস্ট কিভাবে লিখেছে তা দেখুন。

## ৭. লোকাল ডেভেলপমেন্ট স্টেপস
১. ডিপেনডেন্সি ইনস্টল: `npm install`
২. এনভায়রনমেন্ট সেটআপ: `cp .env.example .env`
৩. ডকার ইনফ্রাস্ট্রাকচার স্টার্ট: `docker compose up -d`
৪. ডেটাবেস মাইগ্রেশন: `npm run db:migrate`
৫. সিড ডেটা: `npm run seed`
৬. ডেভ সার্ভার স্টার্ট: `npm run dev`
৭. টেস্ট চালান: `npm test`
